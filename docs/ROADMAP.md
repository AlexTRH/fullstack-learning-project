# Планы работы над приложением

Общий roadmap проекта. Можно использовать как чек-лист и дополнять по ходу работы.

## Текущий фокус: ближайшие улучшения (ветка `feature/error-toasts`)

Фронтенд постов завершён (ветка `feature/posts-frontend` смержена в `development`):
- Лента `/posts`, пагинация, форма создания, удаление; страница поста `/posts/[id]`, редактирование `/posts/[id]/edit`.
- На дашборде — блок «My posts» с превью и ссылкой на ленту.

**Задачи (ближайшие):**

- [x] **Обработка ошибок** — тосты при ошибках API (посты, лента и т.д.).
- [ ] **Состояния загрузки** — скелетоны или спиннеры для ленты постов и для кнопок (submit, delete).
- [ ] **Валидация** — ограничения по длине контента на фронте в соответствии с бэкендом (схемы в `backend/src/posts/schemas/`).
- [ ] **Посты на странице профиля** — на `/users/[id]` показывать посты этого пользователя (фильтр `authorId`).
- [ ] **Единый порт/URL** — в README и скриптах указать актуальный порт бэкенда (5000/5001) или вынести в конфиг.

---

## Среднесрочные идеи (бэкенд + фронт)

- [ ] **Лента только от подписок** — эндпоинт вида `GET /api/posts?following=true` (посты от пользователей, на которых подписан текущий); фронт — переключатель «Все» / «Подписки».
- [ ] **Комментарии к постам** — домен Comment, API (create, list by post, delete), UI под каждым постом.
- [ ] **Лайки** — сущность Like по пост/пользователю, счётчик, кнопка «Нравится» в `PostCard`.
- [ ] **Поиск** — по пользователям (уже есть список?), по тексту постов (бэкенд: фильтр по `content` или полнотекстовый поиск).
- [ ] **Вложения к постам** — бэкенд уже поддерживает `attachments`; фронт: загрузка файлов (например, в S3 или локальное хранилище) и отображение в постах.

## Качество и инфраструктура

- [ ] **Тесты** — E2E (Playwright/Cypress) или интеграционные тесты для критичных сценариев (логин, лента, создание поста).
- [ ] **Линтеры и форматтеры** — единый ESLint/Prettier для всего репозитория (в т.ч. фронт).
- [ ] **CI** — запуск линта и тестов на push/PR (GitHub Actions или аналог).
- [ ] **Деплой** — описание или скрипты для деплоя бэкенда и фронта (Vercel, Railway, и т.д.).
